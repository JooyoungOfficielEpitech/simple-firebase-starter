# Navigation Flow Documentation

## Introduction

### Purpose
This document maps all navigation flows in the Orphi app, providing a comprehensive visual reference for understanding user journeys through the application. The diagrams use ASCII art for clarity and ease of maintenance.

### Core Features
1. **Bulletin Board**: Announcement system where organizations post casting calls and actors apply
2. **Music Player**: Practice studio with A-B loop and metronome features for vocal/choreography practice

### User Types
- **Actors**: Browse announcements, apply to posts, practice with music player
- **Organizations**: Create and manage announcements, review applications

### Navigation Patterns
- **Stack Navigation**: Primary navigation pattern with back button support
- **Tab Navigation**: Bottom tabs for main sections
- **Modal Overlays**: Full-screen modals for forms, notifications, and settings

## 9. Modal & Overlay Patterns

### Full-Screen Modals

**1. NotificationCenterScreen**
- **Trigger**: Bell icon in ScreenHeader
- **Presentation**: Slide up from bottom
- **Dismissal**: Swipe down, back button, or tap outside
- **Content**: Scrollable list of notifications with tap actions
- **Z-index**: Highest (covers all content)

**2. Application Modal**
- **Trigger**: "Apply" button on PostDetailScreen
- **Presentation**: Slide up from bottom (full screen)
- **Dismissal**: Cancel button or back navigation
- **Content**: Multi-field form (role select, message, file uploads)
- **Validation**: Required fields before submit

**3. SettingsScreen**
- **Trigger**: Gear icon in ScreenHeader OR bottom tab
- **Presentation**: Navigate to tab OR modal overlay
- **Content**: App preferences and account management
- **Special**: Theme change triggers curtain animation

### Navigation State Management

**Stack Preservation**:
- Back button always returns to previous screen in stack
- Deep links create new navigation stack with breadcrumb
- Modal dismissal returns to previous screen, not root

**Tab Memory**:
- Each tab maintains its own navigation stack
- Switching tabs preserves scroll position and state
- Tab re-tap scrolls to top (iOS pattern)

## Summary

### Navigation Complexity

**Total Screens**: ~15-20 unique screens
**Primary User Flows**: 2 (Bulletin Board, Music Player)
**Secondary Flows**: Profile/Settings, Notifications

4. **Context-Aware Actions**: Buttons and navigation adapt to user role

6. **User Type Separation**: Actor vs. Organization flows are distinct but share infrastructure
